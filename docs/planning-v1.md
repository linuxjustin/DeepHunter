# Investigation Planning Engine v1

The Investigation Planner is DeepHunter's **decision-making core**.
It determines **what should be tested next** by transforming structured
knowledge into a prioritized, phased investigation plan.

The planner is **completely model-independent** — no AI, no LLM, no
embeddings, no prompts.  Every future AI model will consume plans
generated by this engine.

## Architecture

```
PlannerContext ──→ PlanningPipeline ──→ PlannerResult
                       │
                  ┌────┴────┐
                  │         │
            RuleRegistry  PriorityEngine
                  │
        ┌────┬────┼────┬────┐
        │    │    │    │    │
      Recon  Tech  Auth  Bug  Cloud ...
             Rules Rules Rules  Rules
```

### Core Models

| Model | Purpose |
|-------|---------|
| `InvestigationPlan` | Ordered list of steps + metadata (risk, priority, hours) |
| `InvestigationStep` | A single step with phase, title, priority, risk, cost |
| `ManualTest` | A recommended manual test within a step |
| `PlannerContext` | Structured input to the planner (tech, auth, bugs, etc.) |
| `PlannerResult` | Plan + metrics + warnings |
| `PriorityWeights` | Configurable weight vector for priority scoring |
| `RiskScore` | Composite risk (likelihood × impact / 10) |

### Planning Phases

The plan follows 13 phases in order, matching real research methodology:

```mermaid
graph LR
    RECON --> FINGERPRINT --> FRAMEWORK --> AUTH --> AUTHORIZATION
    AUTHORIZATION --> BUSINESS_LOGIC --> INPUT_VALIDATION --> API
    API --> FILE_UPLOAD --> CLOUD --> PRIV_ESC --> EXPLOIT_CHAIN
    EXPLOIT_CHAIN --> REPORT
```

## Rule Engine

Rules are the core building blocks.  Each rule:

- Has a `name`, `description`, `phase`, and `priority` (lower = evaluated first)
- Implements `evaluate(context: PlannerContext) -> list[InvestigationStep]`
- Is registered in a `RuleRegistry`
- Is independently testable

### Built-in Rules (12 total)

| Rule | Phase | Trigger | Steps |
|------|-------|---------|-------|
| `ReconRule` | RECON | Always | 1 (baseline recon) |
| `TechnologyRule` | FINGERPRINT | Per detected technology | 1 per tech (laravel, django, flask, etc.) |
| `FrameworkDetectionRule` | FRAMEWORK_DETECTION | Frameworks detected | 1 (version/config check) |
| `AuthenticationRule` | AUTHENTICATION_ANALYSIS | Per auth mechanism | 1 per mechanism (JWT, OAuth2, SAML, etc.) |
| `AuthorizationRule` | AUTHORIZATION_ANALYSIS | Always | 1 (RBAC/privilege testing) |
| `BugClassRule` | INPUT_VALIDATION | Per bug class | 1 per class (SQLi, XSS, SSRF, etc.) |
| `EndpointAnalysisRule` | API_ANALYSIS | Interesting endpoints | 1 |
| `BusinessLogicRule` | BUSINESS_LOGIC_ANALYSIS | Business logic observations | 1 |
| `FileUploadRule` | FILE_UPLOAD_ANALYSIS | Upload endpoints | 1 |
| `CloudProviderRule` | CLOUD_ANALYSIS | Per cloud provider | 1 per provider (AWS, Azure, GCP) |
| `PrivilegeEscalationRule` | PRIVILEGE_ESCALATION | Always | 1 |
| `ReportPreparationRule` | REPORT_PREPARATION | Always | 1 |

### Creating Custom Rules

```python
from deephunter.planning import PlanningRule, PlannerContext, InvestigationStep, PlanningPhase

class GraphQLRule(PlanningRule):
    name = "graphql"
    description = "Generates GraphQL testing steps"
    phase = PlanningPhase.API_ANALYSIS
    priority = 45

    def evaluate(self, context: PlannerContext) -> list[InvestigationStep]:
        if not any("graphql" in ep.lower() for ep in context.interesting_endpoints):
            return []

        return [
            InvestigationStep(
                phase=self.phase,
                title="GraphQL API Security Analysis",
                description="Test GraphQL introspection, query depth, batching attacks, and auth bypass.",
                priority_score=0.85,
            )
        ]

# Register
from deephunter.planning import RuleRegistry
registry = RuleRegistry.with_default_rules()
registry.register(GraphQLRule())
```

## Priority Engine

The `PriorityEngine` computes a 0.0–1.0 score from six factors:

| Factor | Default Weight | Source |
|--------|---------------|--------|
| Likelihood | 0.30 | How likely the vulnerability exists (0–10) |
| Impact | 0.25 | Potential impact if confirmed (0–10) |
| Confidence | 0.15 | Confidence in the hypothesis (0–1) |
| Complexity (inverted) | 0.10 | 1.0 - technical difficulty |
| Effort (inverted) | 0.10 | 1.0 - (hours / 8) |
| Reward | 0.10 | Potential value of the finding (0–1) |

Weights are auto-normalized to sum to 1.0.  Custom weights are
supported:

```python
from deephunter.planning import PriorityEngine, PriorityWeights

engine = PriorityEngine(weights=PriorityWeights(
    likelihood=0.4,
    impact=0.3,
    confidence=0.2,
    complexity_inverted=0.1,
    effort_inverted=0.0,
    reward=0.0,
))
```

## Pipeline Stages

1. **Load Context** — Set plan metadata from context; emit `ContextLoadedEvent`
2. **Evaluate Rules** — Run all registered rules; collect step candidates
3. **Prioritize Steps** — Recalculate priority scores using PriorityEngine
4. **Sort and Group** — Order by phase sequence then descending priority
5. **Estimate Cost** — Compute total hours, overall risk, priority

Stage failures are caught gracefully — a failed stage returns a
`PlannerResult` with warnings instead of crashing.

## Events

| Event | When |
|-------|------|
| `PlanStartedEvent` | Planning begins |
| `ContextLoadedEvent` | Context is built |
| `StepGeneratedEvent` | A step candidate is generated |
| `StepPrioritizedEvent` | A step's priority changes |
| `PlanCompletedEvent` | Planning finishes successfully |
| `PlanFailedEvent` | A stage fails |
| `PlanUpdatedEvent` | An existing plan is updated |

```python
from deephunter.planning import Planner
from deephunter.planning.events import PlanCompletedEvent

planner = Planner()
planner.event_bus.subscribe(PlanCompletedEvent, lambda e:
    print(f"Plan done: {e.total_steps} steps in {e.elapsed_seconds:.2f}s")
)
```

## Usage

```python
from deephunter.reasoning.session import InvestigationSession
from deephunter.planning import Planner

# From a session
session = InvestigationSession.new("https://example.com")
session.create_observation("endpoint", description="Login page")
session.state.technology_fingerprint.frameworks = ["django"]
session.state.technology_fingerprint.technologies = [...]

planner = Planner()
result = planner.plan_from_session(session)

for step in result.plan.steps:
    print(f"[{step.phase.value}] {step.title} ({step.priority_score:.2f})")

print(f"Total: {result.plan.total_estimated_hours:.1f}h, "
      f"Risk: {result.plan.risk.overall:.1f}/10")

# Save and load
planner.save_plan(result.plan, "/tmp/plan.json")
loaded = planner.load_plan("/tmp/plan.json")
```

## Extension Points

| Point | How |
|-------|-----|
| **Custom rules** | Subclass `PlanningRule`, implement `evaluate()` |
| **Custom priority weights** | Pass `PriorityWeights` to `PriorityEngine` |
| **Custom rule registry** | Subset of rules via `RuleRegistry` |
| **Context from session** | `PlannerContext.from_session(session)` |
| **Direct context** | Build `PlannerContext` manually for headless use |
| **Event subscribers** | Subscribe to any `PlanningEvent` on `planner.event_bus` |
